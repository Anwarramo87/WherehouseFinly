# ✅ إصلاح التصفية المالية - ملخص التغييرات

## 🔍 المشكلة

كانت البطاقة الإجمالية في صفحة الرواتب **لا تعرض أي أرقام** حتى بعد تشغيل "حساب المسير".

### السبب
```typescript
// ❌ الكود القديم
const payrollData = useMemo(() => {
  if (!reportData?.items?.length) return []; // ← هون المشكلة!
  // ...
}, [reportData, ...]);
```

إذا ما في `reportData.items` (payroll run محسوب)، الكود كان يرجع **array فاضي** `[]`، وبالتالي:
- ❌ **البطاقة الإجمالية** = كل شي صفر
- ❌ **الجدول** = فاضي
- ❌ **لا يمكن عمل تصفية مالية** للمستقيلين

## ✅ الحل

عدلنا الكود ليعرض **preview من الموظفين النشطين** حتى لو ما في payroll run:

```typescript
// ✅ الكود الجديد
const payrollData = useMemo(() => {
  // إذا في payroll run محسوب، استخدم بيانات الباك اند
  if (reportData?.items?.length) {
    return reportData.items.map(...); // نفس المنطق القديم
  }
  
  // ✅ جديد: إذا مافي payroll run، اعمل preview من الموظفين النشطين
  return employees.filter(e => e.status === 'active').map((emp) => {
    // احسب:
    // 1. الراتب المستحق (من الدوام + السلف + الخصومات)
    // 2. المكافآت (من صفحة المكافآت)
    // 3. الخصومات (من صفحة الخصومات)
    // 4. الصافي = المستحق + المكافآت - الخصومات
    // ...
  });
}, [reportData, employees, ...]);
```

## 📊 كيف يتم الحساب الآن

### 1. الراتب المستحق (earnedSalary)
```typescript
// الراتب الإجمالي
calcGross = baseSalary + lumpSumSalary + allowances + ...

// أيام العمل الفعلية
actualWorkDays = 26 - absenceDays // أو من الدوام التلقائي

// التأخير
totalDelayMinutes = lateMinutes + earlyLeaveMinutes

// الراتب المستحق
earnedSalary = calcEarnedSalaryTimeTable(
  calcGross,
  actualWorkDays, 
  totalDelayMinutes
)
```

### 2. المكافآت (bonusesTotal)
```typescript
// من صفحة المكافآت لهاد الشهر
bonusesTotal = Σ (bonusAmount + assistanceAmount)
```

### 3. الخصومات (discountsTotal)
```typescript
// من صفحة الخصومات لهاد الشهر
// تشمل: السلف + العقوبات + الخصومات الأخرى
discountsTotal = Σ (amount) // لكل employeeId في هاد الشهر
```

### 4. الصافي النهائي
```typescript
// الحساب النهائي
netPay = earnedSalary + bonusesTotal - discountsTotal

// مقرب لأقرب ألف
netPayRounded = Math.ceil(netPay / 1000) * 1000

// الفرق
roundingDifference = netPayRounded - netPay
```

## 🔥 ميزات جديدة

### ✅ 1. معاينة الرواتب بدون حساب المسير
الآن تقدر تشوف:
- الرواتب المستحقة لكل موظف
- المكافآت المضافة
- الخصومات المطبقة
- الصافي النهائي

**حتى قبل** ما تشغل "حساب المسير"!

## 🧪 للاختبار

### 1. افتح صفحة الرواتب
```
http://localhost:3000/salaries/payroll
```

### 2. اختر شهر (مثلاً: 2026-06)

### 3. لازم تشوف:
- ✅ البطاقة الإجمالية فيها أرقام (مو أصفار)
- ✅ الجدول فيه موظفين
- ✅ كل موظف عندو:
  - الراتب المستحق
  - المكافآت (إذا في)
  - الخصومات (إذا في)
  - الصافي النهائي

### 4. اختبر التصفية المالية:
- روح على الموظفة المستقيلة (دعاء)
- اضغط على زر "تصفية مالية"
- لازم تشوف:
  - الراتب المستحق من أيام العمل
  - أي مكافآت مستحقة
  - أي خصومات (سلف، عقوبات)
  - الصافي النهائي

## 📝 ملاحظات مهمة

### 1. الفرق بين Preview و Payroll Run

**Preview (بدون حساب المسير):**
- ✅ يعرض الأرقام من البيانات الموجودة
- ✅ يحسب الراتب المستحق من الدوام
- ✅ يجمع المكافآت والخصومات
- ⚠️ مو محفوظ في قاعدة البيانات

**Payroll Run (بعد حساب المسير):**
- ✅ نفس الحسابات
- ✅ **محفوظ في قاعدة البيانات**
- ✅ نهائي ومعتمد
- ✅ يمكن طباعة القسائم

### 2. متى تشغل "حساب المسير"؟
- ✅ بعد ما تتأكد من كل البيانات (دوام، مكافآت، خصومات)
- ✅ في نهاية الشهر
- ✅ قبل طباعة القسائم

### 3. المستقيلين
- ✅ الآن يظهرون في القائمة
- ✅ تقدر تشوف تصفيتهم المالية
- ✅ الحسابات نفسها (راتب + مكافآت - خصومات)
